# The VPS of Eli v5.780 dev

![version](https://img.shields.io/github/v/release/ERITEK/The-VPS-of-Eli?include_prereleases&sort=date)
![debian](https://img.shields.io/badge/debian-12%20%7C%2013-red)

**Один скрипт. Одно меню.**

Всё управляется через меню в терминале. Ничего руками в конфигах ковырять не надо.

```
+=========================+
|     The VPS of Eli      |
|  scrp by ERITEK & Loo1  |
|    Claude (Anthropic)   |
|       v5.780 dev        |
+=========================+
```

VPN, обход блокировок, маскировка трафика, голосовые серверы, прокси для мессенджеров, мониторинг и бэкап. Всё в одном месте, на одном сервере, в пару нажатий. Для Debian VPS, работает и на x86, и на ARM. На русском, без танцев с бубном.

> Это dev-версия. Новое обкатано на стенде, но если что то отвалилось, кидай issue на GitHub.

## Быстрый старт

```bash
curl -fsSL https://raw.githubusercontent.com/ERITEK/The-VPS-of-Eli/main/the_vps_of_eli.sh -o eli.sh
bash eli.sh
```

или

```bash
wget -O eli.sh https://raw.githubusercontent.com/ERITEK/The-VPS-of-Eli/main/the_vps_of_eli.sh
bash eli.sh
```

Дальше скрипт сам проведёт по шагам: что вводить, что выбрать, что нажать.

## Для кого это

Для себя, для друзей, для учёбы и работы. Если коротко:

- **свой VPN**, никаких чужих серверов и логов
- **обход блокировок** для клиентов VPN (discord, ютуб и прочее)
- **маскировка**, чтобы провайдер вообще не видел что у тебя VPN
- **голосовой сервер** для своей тусовки
- **прокси для мессенджеров** (телега, сигнал)
- **всё на одном сервере**, без зоопарка из десяти панелек

## Платформа

| Что нужно | Значение |
| --- | --- |
| ОС | Debian 12 или 13 |
| Архитектура | x86 (amd64) или ARM (arm64/aarch64) |
| Виртуализация | KVM (OpenVZ не пойдёт) |
| Доступ | root |
| RAM | минимум 512 MB, лучше 1 GB и больше |
| Порты | зависит от того, что ставишь (AWG это UDP, 3X-UI это TCP, Signal это 80 и 443) |

## Что умеет

### VPN

| Инструмент | Что делает |
| --- | --- |
| **AmneziaWG** | Основной VPN. Шифрует трафик и маскирует его от DPI. Умеет AWG 1.0, 2.0 и обычный WireGuard. Много интерфейсов сразу, QR-коды для телефона |
| **3X-UI** | Веб-панель для прокси Xray. Протоколы VLESS, VMess, Trojan, Shadowsocks. Трафик косит под обычные HTTPS-сайты |
| **Outline** | Самый простой VPN, от Google Jigsaw. Сделал ключ, скинул другу, он вставил в приложение и всё работает. |

### Обход блокировок и маскировка

Это новое в v5.780. Три разных инструмента под три разные задачи.

| Инструмент | Что делает |
| --- | --- |
| **zapret2** | Обход DPI для клиентов VPN. Если у клиента через твой VPN не открывается discord или ютуб, zapret2 обманывает DPI |
| **wg-obfuscator** | Прячет от провайдера клиента сам факт того, что это WireGuard. Провайдер видит просто непонятный поток, а не VPN |
| **mimic** | Прячет сам факт UDP, заворачивая его в TCP. Нужен там, где провайдер режет или душит весь UDP как класс |

Чем они отличаются и что когда включать, смотри в спойлере ниже.

### Прокси для мессенджеров

| Инструмент | Что делает |
| --- | --- |
| **MTProto** | Прокси для телеги с маскировкой (Fake TLS). Генерит ссылку, нажал и подключился. Можно несколько сразу |
| **SOCKS5** | Универсальный прокси с логином и паролем. Работает с чем угодно |
| **Hysteria 2** | Быстрый прокси на QUIC/UDP. Хорошо тянет на плохих каналах с потерями |
| **Signal Proxy** | TLS-прокси для Signal. Docker плюс Let's Encrypt. Нужен свой домен |

### Голос

| Инструмент | Что делает |
| --- | --- |
| **TeamSpeak 6** | Голосовой сервер. Бинарник с GitHub, systemd, SQLite. Сам перехватывает админ-ключ при первом запуске |
| **Mumble** | Бесплатный голосовой сервер. Лёгкий (около 30 MB RAM), шифрование из коробки |

### Обслуживание

| Инструмент | Что делает |
| --- | --- |
| **Unbound DNS** | Свой DNS для клиентов AmneziaWG. DNS-запросы клиентов не утекают на гугл, а остаются на твоём сервере |
| **Диагностика** | Полная проверка сервера по 21 секции. Есть цветной отчёт в терминале и HTML со светофором |
| **Prayer of Eli** | Аудит стека. Находит расхождения, чинит потерянные файлы, обновляет книгу |
| **Telegram бот** | Шлёт алерт в телегу, если сервис упал, диск забился или память кончилась |
| **Healthcheck** | После каждого reboot сам проверяет и поднимает все сервисы |
| **Бэкап** | Один tar.gz со всеми ключами, конфигами и базами. Скачал, сохранил, спишь спокойно |
| **SSH, UFW, обновления** | Смена порта, ключи, fail2ban, файрвол, обновление всех компонентов |

## Рекомендуемый порядок установки

Если ставишь всё с нуля на свежий сервер:

```
1. Старт, первичная настройка, согласиться на reboot
   (обновит систему, поставит Docker, настроит сеть и безопасность)

2. VPN, AmneziaWG, Установка
   (создаст туннель, выдаст QR-код для телефона)

3. Обслуживание, Unbound DNS
   (свой DNS для VPN, ставить после создания AWG-интерфейса)

4. Обслуживание, Автообслуживание
   (cron reboot, чистка логов, healthcheck)

5. Обслуживание, Telegram мониторинг
   (бот будет слать алерты, если что упадёт)

6. Обход блокировок по необходимости:
   - клиентам не открываются сайты, ставь zapret2
   - провайдер палит WireGuard, ставь wg-obfuscator
   - провайдер режет UDP, ставь mimic

7. VPN, 3X-UI / Outline / Прокси по необходимости

8. Связь, TeamSpeak / Mumble по необходимости

9. Обслуживание, UFW, Включить
   (ВАЖНО: только после настройки всех портов, иначе закроешь себе доступ)

10. Обслуживание, Бэкап, Создать
   (первый снимок настроек, скачай и сохрани)
```

## Известные особенности

1. **UFW сам не включается.** Ты включаешь его вручную, когда все порты настроены, чтобы не закрыть себе SSH.
2. **AWG на кастомном ядре.** Если у VPS кастомное ядро, скрипт ставит стандартное ядро Debian, а после reboot healthcheck сам доустанавливает модуль.
3. **Signal Proxy.** Порты 80 и 443 жёстко. Если 443 занят, сначала освободи его у конфликтующего сервиса.
4. **zapret2 без блока вреден.** Десинк включаем только когда блокировка реально есть. Если сайт и так открывается, zapret2 может наоборот сломать клиента. Проверка блока идёт автоподбором, руками ничего копипастить не надо.
5. **wg-obfuscator и обычный WireGuard.** Обфускатор работает только с vanilla-WG интерфейсом, поэтому он поднимает или использует отдельный WG-интерфейс, не AWG.
6. **mimic только под Linux-клиентов.** Серверная часть ставится на любой VPS, а вот клиент нужен на Linux с ядром 6.1+ и DKMS. Windows, macOS, Android пока мимо.
7. **Telegram бот мониторит только изнутри VPS.** Если сервер лёг целиком, бот молчит (он на том же сервере). Для внешнего мониторинга бери [uptimerobot.com](https://uptimerobot.com).
8. **Unbound слушает только localhost и IP AWG-туннелей.** Outline, 3X-UI, Hysteria свой DNS используют и от Unbound не зависят.
9. **SSH drop-in.** Основной `/etc/ssh/sshd_config` скрипт не трогает. Все его настройки лежат в `/etc/ssh/sshd_config.d/99-eli.conf`. Если что то пошло не так, файл можно удалить и SSH откатится к дефолту.

<details>
<summary><b>AmneziaWG, обфускация подробно</b></summary>

При создании интерфейса выбираешь версию протокола:

| Версия | Параметры | С чем совместимо |
| --- | --- | --- |
| **AWG 1.0** | Jc, Jmin, Jmax, S1, S2, H1-H4 (фиксированные), I1..I5 (CPS) | Keenetic 4.2+, OpenWrt, все клиенты AmneziaVPN |
| **AWG 2.0** | плюс S3, S4, H1-H4 диапазонами | Keenetic 5.1+ dev, свежие AmneziaVPN, OpenWrt с AWG 2.0 (https://github.com/Slava-Shchipunov/awg-openwrt)|
| **WireGuard** | всё обнулено | Любой обычный WG-клиент. Легко палится DPI |

Что значат параметры, если совсем коротко:

- **Jc, Jmin, Jmax** это мусорные пакеты до рукопожатия. Jc это сколько штук, Jmin/Jmax это границы размера.
- **S1, S2** это набивка init/response пакетов рукопожатия.
- **S3, S4** (только AWG 2.0) это набивка cookie и data пакетов.
- **H1-H4** это подписи типа сообщения. Значения 1/2/3/4 это обычный WG (то есть без обфускации), они запрещены, минимум 5.
- **I1..I5** это произвольные пакеты в начале сессии. Маскируют старт туннеля под другой протокол.

#### Пресеты I1..I5

При установке предлагает выбрать, под что косить первые байты сессии. Это то, что DPI увидит самым первым.

| Пресет | Под что косит | Размер | Когда брать |
| --- | --- | --- | --- |
| **DNS** | DNS-запросы к нормальным доменам (пул по регионам: мир, РФ, СНГ, Турция, Иран, Европа, США) | малый | Простые блокировки. Против современного DPI слабоват |
| **STUN** | STUN Binding Request, как у WebRTC | 32-40 байт | Универсальный вариант, косит под видеозвонки |
| **SIP** | SIP INVITE, как у VoIP-клиента (Asterisk, FreeSWITCH, Zoiper и другие) | 285-310 байт | Там, где SIP/VoIP не блокируют. Нужен MTU 1280+ |
| **raw bytes** | Свои hex-байты руками | любой | Для тех, кто знает, что делает |

CPS-теги: `<b 0xHEX>` (фиксированные байты), `<r N>` (рандом), `<rd N>` (рандомные цифры), `<rc N>` (рандомные символы), `<t>` (таймстамп). Тег `<c>` в движке не реализован, поэтому при ручном вводе он блокируется с понятным сообщением.

#### Установка

Через PPA Amnezia, обновляется вместе с `apt upgrade`. Каждый интерфейс хранит свою версию протокола в `iface_*.env`. Клиентский конфиг содержит команду для Keenetic (AWG 1.0) или предупреждение (2.0/WG). QR-код печатается прямо в терминал: навёл камеру, подключился.

<details>
<summary><b>zapret2, обход DPI подробно (новое)</b></summary>

Движок `nfqws2` из проекта bol-van/zapret2. Ставит очередь в nftables и десинкает форвардный трафик клиентов, которые ходят через твои AWG-туннели. Проще говоря: если у клиента через VPN не открывается сайт из за DPI, zapret2 ломает DPI-разбор и сайт открывается.

Важные моменты:

- Работает по SNI (TCP 443) и QUIC initial (UDP 443). DNS не трогает.
- Стратегию обхода подбирает сам (blockcheck). Руками копировать команды не надо.
- Десинк включаем только при подтверждённом блоке. Без блока он скорее навредит.
- Свои nft-таблицы зовутся `zeli_<iface>`, юнит `zapret2-eli@<iface>.service`.
- Живёт мирно рядом с wg-obfuscator и mimic, они на разных плоскостях и не пересекаются.

Обкатано на стенде: Debian 13, ядро 6.12, nfqws2 v1.0.2, форвардный путь на живом клиенте проверен.

</details>

<details>
<summary><b>wg-obfuscator, маскировка WG подробно (новое)</b></summary>

Движок ClusterM/wg-obfuscator, релиз v1.5. Это лёгкий прокси на C в userspace. Ему вообще плевать на ядро, работает даже на OpenVZ/LXC. Прячет от провайдера клиента сам факт того, что это WireGuard.

Как это устроено:

- Обфусцируется отдельный обычный WG-интерфейс (vanilla), не AWG. Скрипт даёт выбрать готовый или создаёт новый.
- Порт этого WG-интерфейса наружу закрывается, снаружи торчит только порт обфускатора.
- У клиента Endpoint переезжает на локальный порт обфускатора, скрипт правит конфиг сам.
- Один инстанс на интерфейс, юнит `wgobfs-eli@<iface>.service`.
- IPv6 не поддерживает. На слабом роутере упирается в CPU, на VPS без проблем.

Обкатано на стенде.

</details>

<details>
<summary><b>mimic, UDP в TCP подробно (новое)</b></summary>

Движок hack3ric/mimic, релиз v0.7.1. Прячет не сигнатуру WireGuard, а сам факт UDP: заворачивает UDP-туннель в настоящий TCP. Нужен там, где провайдер режет или душит весь UDP как класс.

Важные моменты:

- Это модуль ядра через DKMS, ему нужны kernel headers. Скрипт ставит их сам.
- Один инстанс на сетевой интерфейс (WAN), все AWG-порты идут строками в один конфиг.
- Клиенту тоже нужен mimic. Клиент без mimic на этом порту работать не будет, это принцип работы, а не баг.
- Клиент только Linux с ядром 6.1+ и DKMS. Windows, macOS, Android нет.
- Конфиг `/etc/mimic/<wan>.conf`, юнит `mimic@<wan>.service`.

Обкатано на стенде: mimic 0.7.1 ставится и работает.

</details>

<details>
<summary><b>zapret2 или wg-obfuscator или mimic, что когда брать</b></summary>

Три инструмента решают три разные проблемы. Их можно комбинировать.

| Проблема у клиента | Что ставить |
| --- | --- |
| VPN работает, но отдельные сайты не открываются (discord, ютуб) | **zapret2** |
| Провайдер клиента видит и режет сам WireGuard | **wg-obfuscator** |
| Провайдер клиента режет или душит весь UDP | **mimic** |

Коротко: zapret2 чинит сайты за VPN, wg-obfuscator прячет что это WG, mimic прячет что это UDP.

</details>

<details>
<summary><b>MTProto, подробности</b></summary>

Docker-контейнер `nineseconds/mtg:2` (pinned) в режиме Fake TLS. Косит под TLS-трафик к указанному домену (виден в SNI). Генерит ссылку `tg://proxy` с Fake TLS.

Порты по вкусу: 443 (лучшая маскировка), 8443, 993, 5228.

Домен для Fake TLS ищи сам. Дефолт есть, но гарантий нет, DPI регулярно обновляет чёрные списки. Ad tag от бота опционален, прокси работает и без рекламы.

</details>

<details>
<summary><b>Book of Eli, что это и зачем</b></summary>

Центральное JSON-хранилище стека: `/etc/vps-eli-stack/book_of_Eli.json` (chmod 600, директория 700).

Зачем оно:

- **Восстановление.** Если потерялся env-файл или конфиг, в книге записано, что и как было поставлено (версии, пути, порты, флаги). Prayer of Eli по книге чинит дыры.
- **Аудит.** Проверка, что установлено, что включено, что осталось от старого.
- **Дружит с бэкапом.** Один JSON со всей мета-инфой попадает в архив. После переезда на новый VPS книга это единый источник правды.

Функции для работы (из `00_header.sh`):

```bash
book_init                                   # первый запуск
book_read ".system.ssh_port"                # читаем
book_write ".system.ssh_port" "2222" number # пишем (string|number|bool)
book_write_obj ".awg.interfaces.awg0" "$json" # пишем объект
book_del ".zapret.interfaces.awg0"          # удаляем ветку
```

Если jq нет или файл битый, операции тихо пропускаются. Стек работает и без книги, просто аудит не сработает.

</details>

<details>
<summary><b>Prayer of Eli, аудит и восстановление</b></summary>

Запуск: Обслуживание, Prayer of Eli. Сверяет книгу с реальностью и по возможности чинит сам.

| Маркер | Цвет | Что значит |
| --- | --- | --- |
| `[ОК]` | зелёный | Всё совпадает с книгой |
| `[ПОЧИНИЛ]` | зелёный | Нашёл расхождение и исправил |
| `[ОБНОВИЛ]` | голубой | Подтянул в книгу реальные данные (например, версию компонента) |
| `[ВНИМАНИЕ]` | жёлтый | Расхождение есть, но чинить автоматом опасно, просто сообщает |
| `[НЕ СМОГ]` | красный | Пытался починить, не вышло, нужны руки |

Что проверяет: систему (ядро, IP, SSH-порт), AmneziaWG (и модуль ядра после обновления), Outline, 3X-UI, TeamSpeak, Unbound, Mumble, прокси (MTProto, SOCKS5, Hysteria 2, Signal), Telegram-бота, а теперь ещё и zapret2, wg-obfuscator и mimic.

Что чинит сам: поднимает env-файлы из книги, правит права, обновляет версии, ловит призраков (запись в книге есть, а на диске уже нет), пересобирает конфиг mimic из книги, грузит модули ядра.

Чего не трогает: удалённые ключи, ручные правки конфигов, смену IP после переезда. Про это только сообщает.

</details>

<details>
<summary><b>Healthcheck после reboot, механика</b></summary>

Скрипт `/usr/local/bin/eli-healthcheck.sh` запускается через cron с задержкой 90 секунд:

```cron
@reboot sleep 90 ; /usr/local/bin/eli-healthcheck.sh
```

Почему 90 секунд: ядру надо поднять сеть, systemd запустить юниты, Docker подтянуть контейнеры. За меньшее время на холодном VPS не все успевают.

Что делает: каждый включённый сервис проверяет через `systemctl is-active`, упавший поднимает. У AWG проверяет MSS clamping и при необходимости перезапускает интерфейс. Возвращает `ip_forward=1`, если слетел. Поднимает Docker-контейнеры. Если после обновления ядра не грузится модуль AWG, доустанавливает headers и пересобирает через dkms.

Лог: `/var/log/eli-healthcheck.log`. Полный разбор и восстановление это уже Prayer of Eli, а healthcheck это минимум, чтобы стек сам встал после ребута.

</details>

<details>
<summary><b>Telegram бот, все триггеры алертов</b></summary>

Cron-скрипт `/usr/local/bin/eli-tgbot-monitor.sh` крутится каждые N минут (5/15/30/60 на выбор). Молчит, если всё ок.

| Триггер | Условие |
| --- | --- |
| **Сервис упал** | Любой включённый юнит не active: AWG, Docker, 3X-UI, TeamSpeak, Mumble, Unbound, fail2ban, Hysteria 2 |
| **Контейнер остановлен** | Outline, MTProto, SOCKS5 не в статусе running |
| **Диск 80%** | Корневой раздел на 80-89%, предупреждение |
| **Диск 90%+** | Корневой раздел на 90%+, критично |
| **RAM** | Свободно меньше 64 MB |
| **Fail2ban** | Больше 20 текущих банов в sshd, похоже на массовую атаку |

Настройка: сделать бота через [@BotFather](https://t.me/BotFather), взять токен, узнать свой chat_id через [@userinfobot](https://t.me/userinfobot). Скрипт спросит токен, chat_id, интервал и сам разложит cron.

Если VPS лёг целиком, бот не пришлёт (он на том же сервере). Для внешнего мониторинга бери [uptimerobot.com](https://uptimerobot.com).

</details>

<details>
<summary><b>Troubleshooting, типовые проблемы</b></summary>

| Проблема | Что сделать |
| --- | --- |
| **После смены SSH-порта не подключиться** | Из консоли провайдера: `ufw allow NEW/tcp && systemctl restart ssh`. Если совсем плохо: `rm /etc/ssh/sshd_config.d/99-eli.conf && systemctl restart ssh` (откат на 22) |
| **AWG не стартует после `apt upgrade`** | Ядро обновилось, модуль не пересобрался. `dkms autoinstall` или зайди в Prayer of Eli, он сам доустановит headers и соберёт. Потом `systemctl restart awg-quick@awg0` |
| **Keenetic пишет `invalid H1 value`** | Старый KeeneticOS не понимает AWG 1.5/2.0. Ставь firmware 5.1+ из dev-канала или пересоздай клиента под AWG 1.0 |
| **Signal Proxy не стартует, порт 443 занят** | `ss -tlnp \| grep :443`, найди процесс, освободи порт. Signal требует именно 80 и 443 |
| **MTProto работает, Telegram не подключается** | Fake TLS домен в чёрном списке у этого DPI. Пересоздай инстанс с другим доменом или попробуй порт 443 |
| **3X-UI не открывается в браузере** | Меню 3X-UI, Данные для входа, там URL с правильным путём. Проверь UFW: `ufw status` |
| **zapret2 включил, а стало хуже** | Десинк без реального блока вредит. Убедись, что сайт правда блокируется DPI а не забанен по ip. Стратегию подбирает blockcheck сам, дай ему отработать |
| **wg-obfuscator: клиент не коннектится** | Проверь, что интерфейс именно vanilla-WG, а не AWG. Обфускатор с AWG выдаёт мусор. Наружу должен торчать только порт обфускатора |
| **mimic: клиент не видит сервер** | На стороне клиента тоже должен стоять mimic (Linux 6.1+, DKMS). Клиент без mimic на этом порту работать не будет, это норма |
| **`Скрипт уже запущен (lock)`** | Прошлый запуск повис или убит. `rm /var/run/eli-stack.lock` и заново |
| **fail2ban забанил меня самого** | `fail2ban-client set sshd unbanip МОЙ_IP`. Свой IP в whitelist: `/etc/fail2ban/jail.local`, `ignoreip = ...` |

Если ничего не помогло: Обслуживание, Диагностика. Скинь HTML-отчёт автору или открой issue на GitHub.

</details>

<details>
<summary><b>Такой себе changelog v5.780 dev (что изменилось с 4.508)</b></summary>

#### Что нового

- **zapret2 (обход DPI).** Новый раздел меню. Обходит блокировки для клиентов VPN, десинкает форвардный трафик через nfqws2. Стратегию подбирает автоматом (blockcheck), руками копипастить не надо.
- **wg-obfuscator (маскировка WG).** Новый раздел. Прячет от провайдера клиента сам факт WireGuard. Лёгкий userspace-прокси, работает даже на OpenVZ/LXC.
- **mimic (UDP в TCP).** Новый раздел. Прячет сам факт UDP, заворачивая его в TCP. Нужен там, где UDP режут как класс.
- **Диагностика выросла с 18 до 21 секции.** Добавились проверки zapret2, wg-obfuscator и mimic: стоит ли, работает ли, привязано ли. Signal теперь тоже в диагностике прокси.
- **Замер канала переписан целиком.** Раньше было 10 точек по миру, часть уже мёртвая (Дубай отвалился). Теперь пул регионов расширен по просьбам: Финляндия, Швейцария, Германия, Нидерланды, Франция, Испания, США, Россия, Беларусь, Казахстан, Киргизия, Корея, Гонконг. Перед замером идёт лёгкий пробник: мёртвая точка честно пишется как "недоступна", а не как 0 Мбит/с, и не висит по таймауту.
- **Prayer of Eli знает про новые модули.** Сверяет и чинит книгу для zapret2, wg-obfuscator и mimic, ловит призраков, пересобирает конфиг mimic из книги.

#### Что починили и улучшили

- **Единый вид у всех окон установки.** AWG, zapret2, wg-obfuscator и mimic теперь ведут себя одинаково: понятные подсказки, дефолты по смыслу, пауза после установки, чтобы вывод не улетал.
- **Пойман коварный баг с проверкой модулей ядра.** Из за связки `set -o pipefail` и `grep -q` проверка загруженного модуля иногда врала, и mimic/AWG "ставились через раз". Теперь модуль проверяется через `/sys/module/...` без пайпа. Тот же фикс прилетел в AWG и nfqws2.
- **Диагностика: xray-версия ищется по архитектуре.** Раньше был хардкод amd64, на ARM показывало "?". Теперь берётся нужный бинарник.
- **Диагностика: Mumble ловится в обоих вариантах** (mumble-server и murmurd).
- **AmneziaWG подрос:** Расширенные пулы STUN и SIP, правка существующего клиента, временная HTTP-раздача конфига со случайным портом и авто-закрытием.
- Куча мелких фиксов по модулям: чистка мёртвых полей в книге при удалении, аккуратный откат nft, консистентность состояния `.installed`.

</details>
