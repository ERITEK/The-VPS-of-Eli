# The VPS of Eli v3.236

**Один скрипт. Одно меню. Свой VPN, прокси и голосовой сервер на VPS за 10 минут.**

Bash-скрипт для Debian 12/13, который разворачивает полный стек самостоятельного хостинга: **AmneziaWG** (VPN с маскировкой трафика), **3X-UI** (веб-панель Xray для VLESS/VMess/Trojan), **Outline** (Shadowsocks от Google Jigsaw), **MTProto** (прокси для Telegram), **Hysteria 2** (быстрый UDP-прокси), **SOCKS5**, **Signal TLS Proxy**, **TeamSpeak 6**, **Mumble**. Плюс обвязка: Unbound DNS, fail2ban, UFW, healthcheck после reboot, Telegram-мониторинг, бэкап всего стека в один tar.gz.

19 модулей, всё управляется через интерактивное меню.

## Что умеет

### VPN

| Компонент | Что делает | Для |
| --- | --- | --- |
| **AmneziaWG** | Шифрует и маскирует весь трафик. Поддерживает AWG 1.0, AWG 2.0 и обычный WireGuard. Мультиинтерфейс, QR-коды для мобильных клиентов | Основной VPN ежедневного использования |
| **3X-UI** | Веб-панель в браузере для управления прокси Xray. Протоколы VLESS, VMess, Trojan, Shadowsocks. Трафик маскируется под обычные HTTPS-сайты | Продвинутые пользователи |
| **Outline** | Простейший VPN от Google Jigsaw. Генерируешь ключ, отправляешь другу -> он вставляет в приложение и всё работает. Без настроек | 'НЕ раздавай' VPN друзьям и семье |

### Прокси: доступ к мессенджерам

| Компонент | Что делает | Для |
| --- | --- | --- |
| **MTProto** | Прокси для Telegram с маскировкой (Fake TLS). Генерирует ссылку: нажал, подключился. Мультиинстанс, мультисекрет | Telegram в заблокированных регионах |

| **Hysteria 2** | Быстрый прокси на базе QUIC/UDP. Хорошо работает на каналах с потерями пакетов. Мультиинстанс, мультиюзер | Быстрый обход блокировок |
| **Signal Proxy** | TLS прокси для мессенджера Signal. Docker + Let's Encrypt. Требует домен | Signal в заблокированных регионах |

### Голос: свои серверы для общения

| Компонент | Что делает | Для |

| **Mumble** | Бесплатный голосовой сервер. Лёгкий (~30 MB RAM), шифрование из коробки | Когда нужно бесплатно и просто |

### Обслуживание

| Компонент | Что делает |
| --- | --- |
| **Unbound DNS** | Свой DNS-резолвер для клиентов AmneziaWG. DNS-запросы клиентов остаются на твоём сервере |

| **Бэкап** | Один tar.gz со всеми ключами, конфигами, базами. Скачал, сохранил = всё в безопасности |

## Рекомендуемый порядок установки

Если ты ставишь всё с нуля на свежий сервер:

```
1. Старт: первичная настройка: согласиться на reboot
   (обновит систему, поставит Docker, настроит сеть и безопасность)

2. VPN: AmneziaWG: Установка
   (создаст VPN-туннель, выдаст QR-код для телефона)

3. Обслуживание: Unbound DNS
   (свой DNS для VPN, ставить после создания AWG интерфейса)

4. Обслуживание: Автообслуживание
   (настроит cron reboot, очистку логов, healthcheck)

5. Обслуживание: Telegram мониторинг
   (бот будет слать алерты если что-то упадёт)

6. VPN: 3X-UI / Outline / Прокси: по необходимости
   (дополнительные VPN и прокси для мессенджеров)

7. Связь: TeamSpeak / Mumble: по необходимости
   (голосовые серверы)

8. Обслуживание: UFW: Включить
   (ВАЖНО: только после настройки всех портов, иначе заблокируешь себя)

9. Обслуживание: Бэкап: Создать
   (первый снимок всех настроек: скачай и сохрани)
```

## AmneziaWG: подробности

При создании интерфейса выбирается версия протокола:

| Версия | Параметры | Совместимость |
| --- | --- | --- |
| **AWG 1.0** | Jc, Jmin, Jmax, S1, S2, H1-H4 (фиксированные) | Keenetic 4.2+, OpenWrt, все клиенты AmneziaVPN |
| **AWG 1.5** | + I1-I5 (signature chain / CPS) | AmneziaVPN 4.x+ |
| **AWG 2.0** | + ranged H, S3/S4, I1-I5 | Keenetic 5.1+, AmneziaVPN 4.8.12.9+, OpenWrt с пакетами AWG 2.0 |
| **WireGuard** | Всё обнулено, H1-H4 = 1-4 | Любой стандартный WireGuard клиент |

AWG 2.0 добавляет padding для cookie (S3) и data (S4) пакетов, а H1-H4 генерируются как непересекающиеся диапазоны в разных порядках величины: DPI не может зацепиться за статичный header. I1-I5 маскируют хендшейк под реальные протоколы (QUIC, DNS, STUN) -> встроены готовые пресеты.

Установка через PPA Amnezia (обновления через `apt upgrade`). Если ядро VPS кастомное: скрипт ставит стандартное ядро Debian и после reboot автоматически доустанавливает модуль (healthcheck).

Каждый интерфейс хранит свою версию протокола в `iface_*.env` (`AWG_VERSION`).

Клиентские конфиги содержат Keenetic `asc` команду (AWG 1.0) или предупреждение (1.5/2.0/WG). QR-код выводится в терминале: навёл камеру, подключился.

Удаление пиров через **awk**, без python3 -> чистая portable bash-утилита.

## MTProto: подробности

Docker-контейнер `seriyps/mtproto-proxy:0.8.4` с режимом Fake TLS (`-a tls`). Маскируется под TLS-трафик к указанному домену (видно в SNI). Генерирует ссылку `tg://proxy` с `ee`-префиксом (Fake TLS).

Рекомендуемые порты: 443 (лучшая маскировка), 8443, 993, 5228.

Домены для Fake TLS ищите самостоятельно. Дефолт есть, но гарантий нет.

Ad tag от `@MTProxybot` опционален. Если не задан -> используется заглушка (прокси работает без рекламы).

Мультисекрет: один инстанс может хостить несколько секретов (как отдельные пользователи). Отзыв одного секрета не трогает остальных.

## Мониторинг

### Telegram бот

Cron-скрипт проверяет стек каждые N минут (5/15/30/60) и шлёт алерт в Telegram если:

* Любой enabled сервис не active (AWG, Docker, 3X-UI, TS, Mumble, Unbound, fail2ban, Hysteria 2)
* Outline, MTProto или SOCKS5 контейнер остановлен
* Диск заполнен > 80% (предупреждение) или > 90% (критично)
* RAM свободно < 64 MB
* Fail2ban забанил > 20 IP (SSH brute force)

Дедупликация по хэшу: один и тот же алерт не шлётся повторно пока состояние не изменится.

Если VPS целиком лёг -> бот не пришлёт ({л0ги4но} он на том же VPS). Для внешнего мониторинга юзай [uptimerobot.com](https://uptimerobot.com).

### Healthcheck после reboot

Скрипт `/usr/local/bin/eli-healthcheck.sh` запускается через 90 секунд: cron `@reboot sleep 90`:

* Проверяет каждый enabled сервис -> restart если не active
* AWG: MSS clamping в iptables -> рестарт интерфейса если правила не появились
* ip_forward -> включает обратно если слетел
* Outline, MTProto, SOCKS5 контейнеры -> docker start если остановлены
* AWG dkms_fallback: если ядро обновлялось -> доустанавливает модуль amneziawg
* Hysteria 2 -> проверка и рестарт всех инстансов

Лог: `/var/log/eli-healthcheck.log`.

### Book of Eli

Центральное хранилище данных стека. JSON, `/etc/vps-eli-stack/book_of_Eli.json` (600, директория 700).

Функции: `book_init`, `book_read`, `book_write`, `book_write_obj`, `_book_ok`.

Prayer of Eli сверяет книгу с реальным состоянием: проверяет сервисы, пути, версии, восстанавливает потерянные env.

## Бэкап и восстановление

Один tar.gz со всеми данными стека:

| Компонент | Что сохраняется |
| --- | --- |
| Book of Eli | `book_of_Eli.json` |
| AmneziaWG | env, ключи сервера, ключи и конфиги клиентов, `.conf` интерфейсов |
| 3X-UI | env + `x-ui.db` |
| Outline | env + `manager_key.json` |

| MTProto | env инстансов (секрет, порт, домен) |
| SOCKS5 | env инстансов (логин, пароль, порт) |
| Hysteria 2 | конфиг, сертификат, env |

## Известные особенности

1. **AWG 2.0 и роутеры**: Keenetic 4.2 не поддерживает S3/S4 и ranged H из коробки. Нужен Keenetic 5.1+ или свежие модели роутеров.
2. **Signal Proxy**: порты 80 + 443 жёстко. Если 443 занят: сначала смени порт конфликтующего сервиса. Репозиторий Signal-TLS-Proxy архивирован Signal с 2022 -> работает, но обновлений нет.
3. **AWG dkms_fallback**: при кастомном ядре VPS скрипт ставит стандартное ядро, после reboot healthcheck доустанавливает модуль автоматически.
4. **MTProto ad_tag**: образ `seriyps/mtproto-proxy` требует `-t` (ad tag) даже если он не нужен. Скрипт подставляет заглушку из нулей: прокси работает без рекламы.
5. **3X-UI API**: endpoint `/panel/api/inbounds/list` (не `/xui/inbound/list`).
6. **Outline PUT /name**: HTTP 204 = успех, не ошибка.
7. **TeamSpeak 6**: SQLite WAL: при бэкапе копируются 3 файла (.sqlitedb + -shm + -wal).
8. **Fail2ban**: Debian 12+ без `/var/log/auth.log` -> backend=systemd.
9. **UFW**: не включается автоматически. Пользователь включает после настройки всех портов чтобы не заблокировать себе доступ.
10. **Telegram бот**: мониторит только изнутри VPS. Если VPS целиком лёг = бот не работает. Для внешнего мониторинга: [uptimerobot.com](https://uptimerobot.com).
11. **Unbound DNS**: слушает только на IP AWG-туннелей и localhost. Outline, 3X-UI, Hysteria и другие сервисы используют свой собственный DNS и не зависят от Unbound.
12. **GitHub API anonymous limit**: 60 запросов в час (для обновлений 3X-UI, TeamSpeak, Hysteria). Для обычного использования хватит с запасом.
